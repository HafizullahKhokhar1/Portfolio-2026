# Fullstack Portfolio 2026

Premium GitHub-ready portfolio for Hafizullah Khokhar with booking flows, AI chat, certification management, and a polished repository structure.

## Overview

This project is designed as a static fullstack-style portfolio experience for clients, recruiters, and GitHub visitors. It combines a conversion-focused landing page with an admin dashboard for managing certifications and bookings.

## Professional Experience

- Full Stack Developer focused on client-facing portfolio systems
- AI/ML and Agentic AI solution builder
- Automation and workflow integration specialist
- Freelancer-ready experience presentation for direct client outreach

## Highlights

- Modern 2026 portfolio UI with bold gradients and strong typography
- Hero section with portrait, animated signature, and typed service line
- Live GitHub project cards with filters and previews
- Booking system for free consultation, paid sessions, and project intake
- Embedded AI assistant for lead guidance and booking support
- Admin panel for certifications, bookings, and exportable data
- Responsive layout for desktop, tablet, and mobile
- Dark mode with consistent chat visibility and contrast

## API Setup

The project works without APIs by falling back to local behavior.

- Contact form can use Web3Forms if you add an access key
- Client assistant can use a Supabase Edge Function if you deploy one

## Development Notes

- Main source files now live under `assets/css` and `assets/js`
- Documentation is organized under `docs/`
- The profile image remains under `assets/images/`
- `README.md` stays at the root for GitHub visibility

## Updating the Project

When you make changes:

1. Update the source files in `assets/js` and `assets/css`
2. Keep the HTML entry points at the repository root
3. Refresh the docs in `docs/`
4. Commit the changes with a clear message
